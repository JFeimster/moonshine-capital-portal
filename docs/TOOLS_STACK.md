# TOOLS_STACK

## Core stack now
- **Vercel / Next.js** — website, app logic, routing, API routes
- **Tally** — public application and quote forms
- **Notion** — broker CRM / operational source of truth
- **n8n** — automations, webhooks, branching logic, notifications
- **GitHub** — repo, issues, PR workflow, docs, task queue

## Optional later
- **Wix** — optional downstream public CMS / publishing target
- **Google Sheets** — lightweight reporting or logs if needed
- **HubSpot** — CRM or lead-sync if/when useful
- **Dub or Bitly** — link management if you want branded short links later

## Why this stack
It gives you:
- fast deployment
- simple form capture
- flexible automation
- lightweight ops CRM
- room to grow into portal/admin without rebuilding from zero

## Recommended rule
Do not add tools because they look enterprise.
Add them only when they reduce manual ops or unlock revenue.
